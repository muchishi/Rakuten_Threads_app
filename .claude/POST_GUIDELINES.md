# POST_GUIDELINES.md

## 投稿設計の方針

### 投稿タイプの比率

| タイプ | 比率 | 目的 |
|--------|------|------|
| casual（日常・共感） | **70%** | コメント・フォロー獲得・信頼構築 |
| product（商品アフィリエイト） | **30%** | 収益 |

アフィリエイト投稿の連投は「広告アカウント」認定されアルゴリズムに嫌われる。

### Threads アルゴリズムの優先シグナル（重要度順）

| シグナル | 重要度 |
|----------|--------|
| コメント・返信数 | ★★★★★（最重要） |
| 保存数 | ★★★★☆ |
| リポスト・引用 | ★★★★☆ |
| いいね数 | ★★★☆☆ |
| テキストの文脈（本文の言葉） | ★★★★☆ |

→ **ハッシュタグより本文の質を重視。** 大量タグは逆効果。

---

## プロンプトのルール

### やること（必須）

1. **冒頭1行でスクロールを止めるフックを入れる**
   - 数字・損失回避・共感・問いかけのいずれか

2. **体験談・変化・ベネフィットで語る**
   - 「使った後どう変わったか」「周りにどう言われたか」

3. **必ずコメントを誘発する問いかけを1つ入れる**
   - 「あなたはどっち派？」「これわかる人いる？」

4. **リスト形式の場合は保存を促す一言を入れる**
   - 「セール前に保存しておくと便利」

5. **1〜2文ごとに改行してスマホで読みやすくする**

6. **絵文字は3〜5個（商品投稿）/ 2〜3個（カジュアル）**
   - 意味のある箇所のみ。装飾目的は禁止

7. **商品投稿の文末に `#PR（楽天アフィリエイトリンクを含みます）` を必ず入れる**

### やってはいけないこと

| NG | 理由 |
|----|------|
| 商品スペックの羅列 | 感情を動かさない・読み流される |
| 「買ってください」「おすすめします」連発 | 広告感が強く信頼を損なう |
| 誇大表現（「最安！」「絶対！」「奇跡の！」） | 景表法違反リスク |
| ハッシュタグの乱用 | 2026年現在は本文の文脈が重視される |
| 毎回同じ CTA（「プロフから」だけ繰り返す） | 飽きられる |
| PR表記なしのアフィリエイトリンク | 景表法・ステルスマーケティング規制違反 |

## 商品投稿の投稿タイプ判定ロジック

`generate_product_post.py` の `determine_post_type()` が自動判定する。

| 条件 | 投稿タイプ |
|------|-----------|
| レビュー件数 > 500件 | ランキング型 |
| キーワードに「日用品」「キッチン」「掃除」等 | 保存型 |
| それ以外 | 共感型（デフォルト） |

セール期間中（未実装）は「セール型」を優先する予定。

## 投稿タイプ別の書き方

### 共感型（体験談ストーリー）

```
冒頭：「あるある」共感フレーズ or 自己開示
   ↓
体験談：使う前→使った後の変化
   ↓
商品への自然な接続（「実はこれ楽天で買えて」）
   ↓
リンク誘導（「↓リプ欄のリンクから」）
   ↓
CTA：コメント誘発（「同じ悩みある人いたら教えて」）
   ↓
#PR
```

### 保存型（リストまとめ）

```
冒頭：「保存推奨📌」+ リスト予告
   ↓
箇条書き3〜5件（一言コメント付き）
   ↓
信頼補足（「全部実際に使った感想です」）
   ↓
リンク誘導
   ↓
CTA：保存促進（「後で見返す人はとりあえず保存して」）
   ↓
#PR
```

### ランキング型（順位形式）

```
冒頭：「楽天で買ってよかったもの 正直ランキング TOP3」
   ↓
🥇🥈🥉 各1行コメント
   ↓
リンク誘導
   ↓
CTA：「気になる番号コメントで教えて！」
   ↓
#PR
```

### セール型（数字と緊急性）

```
冒頭：「⚠️ 今だけ：〇〇が▲▲%OFF」
   ↓
定価→セール価格の差額
   ↓
商品の魅力1行
   ↓
「スーパーセール終わったら値段戻る」
   ↓
リンク誘導
   ↓
CTA：「今日が一番お得に買えます！」
   ↓
#PR
```

## カテゴリ別の切り口

| カテゴリ | 重点ポイント |
|----------|------------|
| 美容・コスメ | ① Before/After の変化を匂わせる ② 他人評価（「褒められた」） ③ 友達に聞かれる系 |
| 健康・ダイエット | 体の変化・継続できている実感 |
| 食品・飲料 | 味・満足感・家族の反応 |
| 日用品 | 便利さ・時短・Before/After の具体描写 |
| 寝具・インテリア | 快適さ・リラックス・生活の変化 |

## カジュアル投稿のテーマ候補

```python
CASUAL_THEMES = [
    "散歩", "仕事", "映画", "アニメ", "音楽",
    "カフェ", "美容", "読書", "旅行", "食事",
    "買い物", "運動", "季節", "人間関係",
    "朝の習慣", "夜のルーティン", "週末の過ごし方",
    "最近ハマってること", "ちょっとした失敗談", "気づいたこと",
]
```

カジュアル投稿には **PR表記を絶対に入れない**。  
末尾にコメント誘発の問いかけを **必ず1つ** 入れる（Threads アルゴリズム最重要シグナル対策）。

## Gemini システムプロンプト（商品投稿のみ適用）

`config.py` の `GEMINI_SYSTEM_PROMPT` として定義。`generate_with_fallback()` の `system_instruction` 引数に渡す。

主な指示内容：
- 冒頭フックの必須化
- スペック羅列禁止
- コメント誘発 CTA 必須
- 絵文字3〜5個ルール
- 誇大表現禁止
- `#PR` 表記必須
- 投稿タイプ（共感型/保存型/セール型/ランキング型）別ルール

カジュアル投稿はシステムプロンプトなし（役割が異なるため混在を避ける）。
