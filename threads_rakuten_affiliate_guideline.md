## 1. リサーチ結果：アカウント分析

### 成功アカウントの共通プロフィール設計

| 項目 | 傾向 |
|------|------|
| フォロワー数 | 開始1ヶ月で500〜5,000人。フォロワー10人以下でも収益化した事例あり |
| 投稿頻度 | 1日2〜5投稿。「毎日必ず投稿」より「高品質を週5」が2026年トレンド |
| 投稿ジャンル | 単一ジャンル特化（育児・美容・ガジェット・日用品など）が圧倒的に有利 |
| プロフィール設計 | 「誰のためのアカウントか」「何が得られるか」を1行で明記 |
| CTA特徴 | プロフィールリンクへの誘導 or コメント促進の2択が多い |

### 投稿ジャンル別の割合（成功アカウント平均）

| 種別 | 割合 | 目的 |
|------|------|------|
| アフィリエイト商品紹介投稿 | **30%以下** | 収益 |
| お役立ち情報・まとめ | **40%** | 保存・拡散・信頼構築 |
| 共感・日常・問いかけ | **30%** | コメント・フォロー獲得 |

> ⚠️ アフィリエイト投稿だけを繰り返すと「広告アカウント」認定されアルゴリズムから嫌われる

### プロフィール文の構造（バズアカウントの型）

```
【アイコン】一目でジャンルがわかるもの
【名前】ジャンル名 or ターゲット層を含む名前（例：楽天まとめ｜ズボラ主婦の節約術）
【自己紹介文】

  └ 何が得られるか（例：「毎日お得な楽天情報を届けます」）
```

## 2. リサーチ結果：投稿分析

### バズりやすい投稿の構成パターン

#### パターンA：「情報まとめ型」（保存狙い）
```
冒頭フック（1行）
↓
リスト形式の情報（3〜7個）
↓
締め：「保存して後で見返して」+ 問いかけ
↓
アフィリンク or 「プロフィールリンクから」
```

#### パターンB：「共感エピソード型」（いいね・コメント狙い）
```
冒頭：「あるある」ストーリー（1〜2行）
↓
問題提起（「こんなこと困ってませんか？」）
↓
解決策＋商品の自然な登場
↓
「実際に使ってみたら〜だった」体験談
↓
商品リンクへの誘導
```

#### パターンC：「セール訴求型」（購買促進）
```
緊急性フック（「今だけ！」「あと〇時間！」）
↓
具体的な割引率・金額
↓
商品説明（1〜2行のみ）
↓
購入した時のベネフィット
↓
リンク誘導
```

#### パターンD：「ランキング型」（拡散・保存狙い）
```
「〇〇おすすめ TOP3」など
↓
順位と商品名・一言コメント
↓
「あなたはどれが好き？」などの問いかけ
↓
「詳しくはプロフから」
```

### 冒頭フックの分析

インプレッションが高い投稿は**最初の1行でスクロールを止めている**。共通パターン：

- **数字で始める**：「楽天で3000円以下の〇〇TOP5」
- **問いかけ**：「まだ〇〇に△△円払ってるの？」
- **否定形の驚き**：「これ知らないと損します」
- **限定性**：「今だけ●%OFF」
- **共感フレーズ**：「育児で疲れた夜にこれ頼んだら正解だった」

### 改行・絵文字パターン

```
【改行】1〜2文ごとに改行（スマホで読みやすいよう）
【絵文字】1投稿3〜5個まで。意味のある箇所のみ（装飾目的NG）
　✅ 良い使い方：「💰お得情報」「⚠️注意点」「✨おすすめ」
　❌ 悪い使い方：「🌸💕✨🌈💖」（意味なし装飾）
```

### コメント誘発の手法

| 手法 | 例文 |
|------|------|
| 2択質問 | 「○○派？△△派？コメントで教えて！」 |
| 経験質問 | 「皆さんはこれ使ったことありますか？」 |
| 共感募集 | 「これわかる人いる？笑」 |
| 意見募集 | 「このジャンルでもっと知りたいことある？」 |
| ランキング投票 | 「1位から順に気になる番号コメントして」 |

### 保存を促す手法

- 「後で買うリスト」「セール前チェックリスト」として提示
- 「保存推奨」を本文中に自然に入れる（命令形より「参考になったら」）
- リスト形式・まとめ形式で「何度も見返したくなる」設計

### 商品リンクへの誘導パターン

| タイプ | 文例 | 備考 |
|--------|------|------|
| **ツリー型（リプ欄）** | 「↓リプ欄に価格・レビュー・楽天リンクまとめてます」 | **本プロジェクト採用方式** |
| プロフ誘導型 | 「詳細はプロフのリンクから見てね」 | プロフ遷移が1ステップ増える |
| 直接リンク型 | 「↓商品リンク貼っておきます（r10.to/〜）」 | 本文に外部リンクを含むとリーチが抑制される傾向あり |
| コメント型 | 「欲しい！って人はコメントしてくれたらDMします」 | 手動運用向け（自動化に不向き） |

> ⚠️ **重要（2026年7月時点）**：Threadsは本文に外部リンクを含む投稿のリーチを抑制する傾向が強い。
> 本プロジェクトでは**メイン投稿にはリンクを入れず、リプライ（ツリー）に楽天リンク＋価格＋レビューをまとめて投稿する方式**を採用している（`generate_product_post.py` の `build_reply_text()`）。
> その際、本文末尾で「リプ欄に何があるか」を具体的に予告するとクリック率が上がる。
> - 良い例：「↓リプ欄に価格・レビュー・楽天リンクまとめてます」
> - 悪い例：「↓リプ欄から」だけ（何があるかわからないのでクリックされない）

## 3. 共通点の抽出

### ✅ 成功している投稿の必須要素

| 要素 | 説明 |
|------|------|
| **強い冒頭フック** | 最初の1行で「自分事」と感じさせる |
| **具体的な数字** | 割引率・価格・個数・ランキング順位など |
| **セール連動** | スパセ・マラソン・0のつく日など楽天セールと連動 |
| **損失回避の訴求** | 「これを知らないと損」「買わないともったいない」 |
| **共感ストーリー** | 使ってみた体験・失敗談・あるある話 |
| **問いかけ（CTA）** | コメント・保存・フォローを自然に促す |
| **情報の希少性** | 「実は知られていない〜」「楽天民だけ得する〜」 |
| **読みやすい構成** | 短文改行・絵文字適度使用・リスト形式 |

### ❌ 避けるべき要素

| 要素 | 理由 |
|------|------|
| 商品スペックの羅列 | 感情を動かさない。読み流される |
| 「広告感」が強い文体 | 「買ってください」「おすすめします」連発はNG |
| 商品リンクだけの投稿 | アルゴリズムにスパム判定される恐れあり |
| ハッシュタグの乱用 | 2026年現在、文脈テキストを重視。大量タグは逆効果 |
| 毎投稿同じCTA | 「プロフから」だけを毎回繰り返すと飽きられる |
| 根拠なしの断定表現 | 「絶対おすすめ！」「これ一択！」は信頼を損なう |
| 誇大表現 | 「最安！」「奇跡の〇〇！」は景表法違反リスク |
| アフィ投稿の連投 | 1日のアフィ投稿は3件以下が安全ライン |

## 4. Threadsアルゴリズム考察（2026年最新）

### アルゴリズムが評価するシグナル（重要度順）

| シグナル | 重要度 | 施策 |
|----------|--------|------|
| **コメント・返信数** | ★★★★★ | 問いかけ・2択・意見募集を入れる |
| **保存数** | ★★★★☆ | リスト・まとめ形式で「後で使える」投稿に |
| **リポスト・引用** | ★★★★☆ | 共感・驚きを生む投稿設計 |
| **いいね数** | ★★★☆☆ | エンゲージメントの補助指標 |
| **投稿頻度（アクティブ性）** | ★★★☆☆ | 毎日何らかの投稿を維持 |
| **テキスト内容の文脈** | ★★★★☆ | ハッシュタグより本文の言葉を重視 |
| **アカウント信頼度** | ★★★☆☆ | 過去の投稿パフォーマンスの蓄積 |

### Threadsアルゴリズムの3大特性

1. **フォロワー数より「投稿の質」**
   → フォロワー0でも良質な投稿は数万人に届く。逆に低品質は届かない

2. **会話が生まれた投稿を優先拡散**
   → コメントが来たら「さらに返信したくなる問いかけ」で返すと連鎖拡散

3. **Instagramとのデータ連携**
   → Instagram側でよくいいね・保存してくれるユーザーの興味関心データが初期表示に影響

### セール期間の活用戦略

```
【楽天セールカレンダー】
・楽天スーパーセール（3・6・9・12月）
・お買い物マラソン（毎月1回程度）
・0のつく日（10・20・30日）
・5のつく日（5・15・25日）
・39ショップの日、ポイント5倍デーなど

【戦略】セール2〜3日前から予告投稿 → 当日に購入リンク投稿
```

## 5. 投稿作成ガイドライン（本体）

---

### 【投稿の目的】

この投稿は「商品紹介」ではなく、以下の優先順位で設計する：

```
1位：コメント・返信を生む（アルゴリズム最優先シグナル）
2位：保存される（リピート閲覧・後で買う）
3位：プロフィール遷移・フォロー獲得
4位：リンククリック → 楽天購入
```

**重要原則：「売る前に価値を渡す」**
→ 有益情報・共感・お得感を先に提供し、その流れで商品リンクへ自然誘導する

---

### 【投稿構成の基本フレーム】

```
━━━━━━━━━━━━━━
① 冒頭フック（1〜2行）
　 → スクロールを止める一言。数字・疑問・共感・損失回避
━━━━━━━━━━━━━━
② ボディ（本文 / 3〜7行）
　 → 価値提供。リスト・体験談・比較・ランキングなど
━━━━━━━━━━━━━━
③ 商品への自然な接続（1〜2行）
　 → 「実はこれ楽天で買える」「使ってみた感想が〜」
━━━━━━━━━━━━━━
④ リプ欄予告（1行）
　 → 「↓リプ欄に価格・レビュー・楽天リンクまとめてます」
　 　（本文に直接リンクは入れない。リンクはリプライ投稿に分離）
━━━━━━━━━━━━━━
⑤ CTA（1行）
　 → コメント誘発 or 保存促進 or フォロー促進
━━━━━━━━━━━━━━
⑥ PR表記（必須）
　 → 「#PR（楽天アフィリエイトリンクを含みます）」
━━━━━━━━━━━━━━

【文字数制限】本文は #PR 表記を含めて500文字以内（Threads APIの上限）

【リプライ投稿（ツリー）の構成】
▼ 楽天で詳細・購入はこちら（PR・楽天アフィリエイトリンク含む）
⭐ レビュー評価（件数）
💴 価格
商品URL（アフィリエイトリンク）
```

---

### 【NG例 vs 良い例】

#### ❌ NG例（商品紹介型・広告感が強い）

```
【商品名】〇〇シャンプー 500ml

成分：△△配合
容量：500ml
価格：1,980円（税込）
ブランド：〇〇

乾燥が気になる方におすすめです。
楽天での購入はこちら↓
r10.to/xxxxx

#楽天 #シャンプー #おすすめ #美容 #ヘアケア
```

**問題点：**
- スペック羅列で感情を動かさない
- 「誰の悩みが解決するか」が不在
- コメントも保存も促せていない
- 広告感が強く信頼を損なう
- ハッシュタグの乱用

---

#### ✅ 良い例（共感＋価値提供＋自然誘導型）

```
美容院でいつも褒められる髪の秘密、教えます。

実は市販シャンプーに3000円以上出すなら
これ一本に変えた方がいい理由があって。

・洗った後にきしみが残らない
・翌朝の寝癖が段違いにまとまる
・美容院帰りの質感が3〜4日続く

今まで6本以上試して
やっと出会えた「当たり」です。

しかもセールで40%OFF中なので
今買わないともったいないレベル。

↓リプ欄に価格・レビュー・楽天リンクまとめてます🔗

あなたが今使ってるシャンプー、
満足してる？教えてほしいな👇

#PR（楽天アフィリエイトリンクを含みます）
```

**良い点：**
- 冒頭で「自分事感」を演出
- 具体的な体験談でリアリティ
- 箇条書きで読みやすく保存したくなる
- 限定性（セール）で購買緊急性
- 問いかけでコメントを自然に誘発

### 【投稿設計の黄金比率（自動運用）】

本プロジェクトは GitHub Actions で **3時間ごとに1投稿**（1日最大8投稿）を自動実行する。
投稿タイプは `run.py` が確率で決定する。

```
casual（日常・共感投稿）：コメント・フォロー獲得・信頼構築
product（商品アフィリエイト投稿）：収益

推奨比率：casual 70% / product 30%
→ アフィリエイト投稿が30%を超えると「広告アカウント」認定リスクが上がる
→ 1日8投稿なら product は2〜3本以内に収まる計算
```

**casual投稿の設計（`generate_casual_post.py`）：**
- テーマは `config.py` の `CASUAL_THEMES`（約20種）からGeminiが選択。直近20件の使用テーマは除外
- 季節ヒント（月ごとの話題）をプロンプトに注入し、時期に合った内容にする
- 末尾にコメント誘発の問いかけを必ず1つ入れる（最重要アルゴリズムシグナル対策）
- **PR表記・商品紹介は絶対に入れない**（casualの役割は信頼構築）
- 丁寧語を避けたSNSらしいカジュアル文体、絵文字2〜3個

## 6. Geminiへ渡すシステムプロンプト

システムプロンプトは `config.py` の `GEMINI_SYSTEM_PROMPT` として定数化されており、
商品投稿（product）の生成時のみ `generate_with_fallback()` の `system_instruction` 引数に渡される。
**カジュアル投稿にはシステムプロンプトを使わない**（役割が異なるため混在を避ける）。

### ✅ 現行システムプロンプト（`config.py` と同期）

```
あなたはThreads（スレッズ）と楽天アフィリエイトのプロフェッショナルです。

【あなたの役割】
入力された楽天市場の商品情報をもとに、
Threadsでインプレッション・コメント・保存・プロフィール遷移を
最大化する投稿文を生成してください。

【Threadsアルゴリズムの前提知識】
・コメント・返信が多い投稿ほどアルゴリズムに評価される
・保存されることでリピート表示とシグナル強化につながる
・フォロワー外にもリーチする設計のため「初見の人でも価値を感じる」必要がある
・ハッシュタグより本文テキストの文脈が重視される

【必ず守るルール】
1. 冒頭1行は「スクロールを止める」フックにする
   （数字・損失回避・共感・問いかけのいずれかを使用）
2. 商品スペックの羅列は禁止。代わりに「使った後の変化・体験談・ベネフィット」で語る
3. 本文中に必ずコメントを誘発する問いかけを1つ入れる
4. リスト形式（箇条書き）を含む場合は保存を促す一言を入れる
5. 1〜2文ごとに改行してスマホで読みやすくする
6. 絵文字は1投稿3〜5個まで。装飾目的の絵文字は禁止
7. 文末に必ず「#PR（楽天アフィリエイトリンクを含みます）」を付ける
8. 誇大表現（「最安！」「絶対！」「奇跡の！」）は禁止
9. 投稿タイプを指示された場合は以下に従う：
   - 共感型：体験談ストーリーで感情に訴える
   - 保存型：リスト形式でまとめ情報を提供
   - セール型：数字と緊急性を前面に出す
   - ランキング型：順位形式で問いかけを末尾に入れる
10. 投稿本文は必ず500文字以内に収めること（Threads API の上限）
11. リンク誘導は「↓リプ欄のリンクから」で統一し、その前に「価格・レビューをリプ欄に貼ってるのでチェックして」など価値の予告を入れると効果的
```

> 旧版にあった「---冒頭フック---」等のセクションラベル付き出力フォーマットは廃止。
> 現行はユーザープロンプト側で「本文のみを出力（セクションラベル・余分な説明不要）」と指示している。

### ✅ Gemini API呼び出し（現行実装：`google-genai` パッケージ）

> ⚠️ 旧版の `google.generativeai`（`genai.configure` / `GenerativeModel`）は**非推奨パッケージ**のため使用しない。
> 現行は `google-genai` パッケージ（`from google import genai`）＋ Gemini 2.5 系モデルを使用する。

```python
# gemini_client.py（現行実装の要点）
from google import genai
from google.genai import types

client = genai.Client(api_key=GEMINI_API_KEY)

config = types.GenerateContentConfig(system_instruction=GEMINI_SYSTEM_PROMPT)
res = client.models.generate_content(
    model="gemini-2.5-flash-lite",  # プライマリ（コスト優先）
    contents=user_prompt,
    config=config,
)
text = res.text
```

**モデル構成とフォールバック（`config.py` / `gemini_client.py`）：**

```
GEMINI_PRIMARY_MODEL  = "gemini-2.5-flash-lite"   # コスト優先（6回リトライ）
GEMINI_FALLBACK_MODEL = "gemini-2.5-flash"        # 高品質（10回リトライ）

プライマリ失敗 → フォールバックへ自動切替 → 両方失敗で Exception 送出
```

実際の呼び出しは共通ラッパー `generate_with_fallback(prompt, system_instruction=...)` を使う。

### ✅ 投稿タイプ自動判定ロジック（`generate_product_post.py` と同期）

```python
def determine_post_type(item: dict) -> str:
    """商品情報から最適な投稿タイプを判定する"""
    review_count = int(item.get("review_count") or 0)
    if review_count > 500:
        return "ランキング型"

    keyword = (item.get("keyword") or "").lower()
    daily_keywords = ["日用品", "キッチン", "掃除", "洗剤", "生活雑貨"]
    if any(k in keyword for k in daily_keywords):
        return "保存型"

    return "共感型"
```

| 条件 | 投稿タイプ |
|------|-----------|
| レビュー件数 > 500件 | ランキング型 |
| 検索キーワードに「日用品」「キッチン」「掃除」「洗剤」「生活雑貨」 | 保存型 |
| それ以外 | 共感型（デフォルト） |

> セール型は未実装。セール期間検知（スーパーセール・お買い物マラソン等）を実装した際に
> `determine_post_type()` の先頭で「セール型」を優先返却する予定。
