## 2. リサーチ結果：投稿分析

### バズりやすい投稿の構成パターン

#### パターンA：「情報まとめ型」（保存狙い）
```
冒頭フック（1行）
↓
リスト形式の情報（3〜7個）
↓
締め：「保存して後で見返して」+ 問いかけ
↓
アフィリンク or 「プロフィールリンクから」
```

#### パターンB：「共感エピソード型」（いいね・コメント狙い）
```
冒頭：「あるある」ストーリー（1〜2行）
↓
問題提起（「こんなこと困ってませんか？」）
↓
解決策＋商品の自然な登場
↓
「実際に使ってみたら〜だった」体験談
↓
商品リンクへの誘導
```

#### パターンC：「セール訴求型」（購買促進）
```
緊急性フック（「今だけ！」「あと〇時間！」）
↓
具体的な割引率・金額
↓
商品説明（1〜2行のみ）
↓
購入した時のベネフィット
↓
リンク誘導
```

#### パターンD：「ランキング型」（拡散・保存狙い）
```
「〇〇おすすめ TOP3」など
↓
順位と商品名・一言コメント
↓
「あなたはどれが好き？」などの問いかけ
↓
「詳しくはプロフから」
```

### 冒頭フックの分析

インプレッションが高い投稿は**最初の1行でスクロールを止めている**。共通パターン：

- **数字で始める**：「楽天で3000円以下の〇〇TOP5」
- **問いかけ**：「まだ〇〇に△△円払ってるの？」
- **否定形の驚き**：「これ知らないと損します」
- **限定性**：「今だけ●%OFF」
- **共感フレーズ**：「育児で疲れた夜にこれ頼んだら正解だった」

### 改行・絵文字パターン

```
【改行】1〜2文ごとに改行（スマホで読みやすいよう）
【絵文字】1投稿3〜5個まで。意味のある箇所のみ（装飾目的NG）
　✅ 良い使い方：「💰お得情報」「⚠️注意点」「✨おすすめ」
　❌ 悪い使い方：「🌸💕✨🌈💖」（意味なし装飾）
```

### コメント誘発の手法

| 手法 | 例文 |
|------|------|
| 2択質問 | 「○○派？△△派？コメントで教えて！」 |
| 経験質問 | 「皆さんはこれ使ったことありますか？」 |
| 共感募集 | 「これわかる人いる？笑」 |
| 意見募集 | 「このジャンルでもっと知りたいことある？」 |
| ランキング投票 | 「1位から順に気になる番号コメントして」 |

### 保存を促す手法

- 「後で買うリスト」「セール前チェックリスト」として提示
- 「保存推奨」を本文中に自然に入れる（命令形より「参考になったら」）
- リスト形式・まとめ形式で「何度も見返したくなる」設計

### 商品リンクへの誘導パターン

| タイプ | 文例 |
|--------|------|
| 直接リンク型 | 「↓商品リンク貼っておきます（r10.to/〜）」 |
| プロフ誘導型 | 「詳細はプロフのリンクから見てね」 |
| コメント型 | 「欲しい！って人はコメントしてくれたらDMします」 |
| ツリー型 | 「↓続きのリンクはリプ欄に！」 |

## 3. 共通点の抽出

### ✅ 成功している投稿の必須要素

| 要素 | 説明 |
|------|------|
| **強い冒頭フック** | 最初の1行で「自分事」と感じさせる |
| **具体的な数字** | 割引率・価格・個数・ランキング順位など |
| **セール連動** | スパセ・マラソン・0のつく日など楽天セールと連動 |
| **損失回避の訴求** | 「これを知らないと損」「買わないともったいない」 |
| **共感ストーリー** | 使ってみた体験・失敗談・あるある話 |
| **問いかけ（CTA）** | コメント・保存・フォローを自然に促す |
| **情報の希少性** | 「実は知られていない〜」「楽天民だけ得する〜」 |
| **読みやすい構成** | 短文改行・絵文字適度使用・リスト形式 |

### ❌ 避けるべき要素

| 要素 | 理由 |
|------|------|
| 商品スペックの羅列 | 感情を動かさない。読み流される |
| 「広告感」が強い文体 | 「買ってください」「おすすめします」連発はNG |
| 商品リンクだけの投稿 | アルゴリズムにスパム判定される恐れあり |
| ハッシュタグの乱用 | 2026年現在、文脈テキストを重視。大量タグは逆効果 |
| 毎投稿同じCTA | 「プロフから」だけを毎回繰り返すと飽きられる |
| 根拠なしの断定表現 | 「絶対おすすめ！」「これ一択！」は信頼を損なう |
| 誇大表現 | 「最安！」「奇跡の〇〇！」は景表法違反リスク |
| アフィ投稿の連投 | 1日のアフィ投稿は3件以下が安全ライン |

## 4. Threadsアルゴリズム考察（2026年最新）

### アルゴリズムが評価するシグナル（重要度順）

| シグナル | 重要度 | 施策 |
|----------|--------|------|
| **コメント・返信数** | ★★★★★ | 問いかけ・2択・意見募集を入れる |
| **保存数** | ★★★★☆ | リスト・まとめ形式で「後で使える」投稿に |
| **リポスト・引用** | ★★★★☆ | 共感・驚きを生む投稿設計 |
| **いいね数** | ★★★☆☆ | エンゲージメントの補助指標 |
| **投稿頻度（アクティブ性）** | ★★★☆☆ | 毎日何らかの投稿を維持 |
| **テキスト内容の文脈** | ★★★★☆ | ハッシュタグより本文の言葉を重視 |
| **アカウント信頼度** | ★★★☆☆ | 過去の投稿パフォーマンスの蓄積 |

### Threadsアルゴリズムの3大特性

1. **フォロワー数より「投稿の質」**
   → フォロワー0でも良質な投稿は数万人に届く。逆に低品質は届かない

2. **会話が生まれた投稿を優先拡散**
   → コメントが来たら「さらに返信したくなる問いかけ」で返すと連鎖拡散

3. **Instagramとのデータ連携**
   → Instagram側でよくいいね・保存してくれるユーザーの興味関心データが初期表示に影響

### セール期間の活用戦略

```

・楽天スーパーセール（3・6・9・12月）
・お買い物マラソン（毎月1回程度）
・0のつく日（10・20・30日）
・5のつく日（5・15・25日）

【戦略】セール2〜3日前から予告投稿 → 当日に購入リンク投稿
```

## 5. 投稿作成ガイドライン（本体）

---

### 【投稿の目的】

この投稿は「商品紹介」ではなく、以下の優先順位で設計する：

```
1位：コメント・返信を生む（アルゴリズム最優先シグナル）
2位：保存される（リピート閲覧・後で買う）
3位：プロフィール遷移・フォロー獲得
4位：リンククリック → 楽天購入
```

**重要原則：「売る前に価値を渡す」**
→ 有益情報・共感・お得感を先に提供し、その流れで商品リンクへ自然誘導する

---

### 【投稿構成の基本フレーム】

```
━━━━━━━━━━━━━━
① 冒頭フック（1〜2行）
　 → スクロールを止める一言。数字・疑問・共感・損失回避
━━━━━━━━━━━━━━
② ボディ（本文 / 3〜7行）
　 → 価値提供。リスト・体験談・比較・ランキングなど
━━━━━━━━━━━━━━
③ 商品への自然な接続（1〜2行）
　 → 「実はこれ楽天で買える」「使ってみた感想が〜」
━━━━━━━━━━━━━━
④ リンク誘導（1行）
　 → 直接リンク or「プロフのリンクから」
━━━━━━━━━━━━━━
⑤ CTA（1行）
　 → コメント誘発 or 保存促進 or フォロー促進
━━━━━━━━━━━━━━
⑥ PR表記（必須）
　 → 「#PR」「楽天アフィリエイトリンクを含みます」
━━━━━━━━━━━━━━
```

---

### 【NG例 vs 良い例】

#### ❌ NG例（商品紹介型・広告感が強い）

```
【商品名】〇〇シャンプー 500ml

成分：△△配合
容量：500ml
価格：1,980円（税込）
ブランド：〇〇

乾燥が気になる方におすすめです。
楽天での購入はこちら↓
r10.to/xxxxx

#楽天 #シャンプー #おすすめ #美容 #ヘアケア
```

**問題点：**
- スペック羅列で感情を動かさない
- 「誰の悩みが解決するか」が不在
- コメントも保存も促せていない
- 広告感が強く信頼を損なう
- ハッシュタグの乱用

---

#### ✅ 良い例（共感＋価値提供＋自然誘導型）

```
美容院でいつも褒められる髪の秘密、教えます。

実は市販シャンプーに3000円以上出すなら
これ一本に変えた方がいい理由があって。

・洗った後にきしみが残らない
・翌朝の寝癖が段違いにまとまる
・美容院帰りの質感が3〜4日続く

今まで6本以上試して
やっと出会えた「当たり」です。

しかもセールで40%OFF中なので
今買わないともったいないレベル。

詳しくはプロフのリンクから見てみて🔗

あなたが今使ってるシャンプー、
満足してる？教えてほしいな👇

#PR（楽天アフィリエイトリンクを含みます）
```

**良い点：**
- 冒頭で「自分事感」を演出
- 具体的な体験談でリアリティ
- 箇条書きで読みやすく保存したくなる
- 限定性（セール）で購買緊急性
- 問いかけでコメントを自然に誘発

## 6. Geminiへ渡すシステムプロンプト

### ✅ 汎用版システムプロンプト（全カテゴリ対応）

```
あなたはThreads（スレッズ）と楽天アフィリエイトのプロフェッショナルです。

【あなたの役割】
入力された楽天市場の商品情報をもとに、
Threadsでインプレッション・コメント・保存・プロフィール遷移を
最大化する投稿文を生成してください。

【Threadsアルゴリズムの前提知識】
・コメント・返信が多い投稿ほどアルゴリズムに評価される
・保存されることでリピート表示とシグナル強化につながる
・フォロワー外にもリーチする設計のため「初見の人でも価値を感じる」必要がある
・ハッシュタグより本文テキストの文脈が重視される

【必ず守るルール】
1. 冒頭1行は「スクロールを止める」フックにする
   （数字・損失回避・共感・問いかけのいずれかを使用）
2. 商品スペックの羅列は禁止。代わりに「使った後の変化・体験談・ベネフィット」で語る
3. 本文中に必ずコメントを誘発する問いかけを1つ入れる
4. リスト形式（箇条書き）を含む場合は保存を促す一言を入れる
5. 1〜2文ごとに改行してスマホで読みやすくする
6. 絵文字は1投稿3〜5個まで。装飾目的の絵文字は禁止
7. 文末に必ず「#PR（楽天アフィリエイトリンクを含みます）」を付ける
8. 誇大表現（「最安！」「絶対！」「奇跡の！」）は禁止
9. 投稿タイプを指示された場合は以下に従う：
   - 共感型：体験談ストーリーで感情に訴える
   - 保存型：リスト形式でまとめ情報を提供
   - セール型：数字と緊急性を前面に出す
   - ランキング型：順位形式で問いかけを末尾に入れる

【出力フォーマット】
以下の構造で出力する：

---冒頭フック---
（スクロールを止める1〜2行）

---本文---
（価値提供・体験談・ベネフィット・リストなど）

---商品接続---
（自然な商品紹介1〜2行）

---リンク誘導---
（プロフリンクへの誘導 or 直接リンク挿入箇所）

---CTA---
（コメント促進 or 保存促進 or フォロー促進のいずれか1つ）

---PR表記---
#PR（楽天アフィリエイトリンクを含みます）

---

【入力情報の使い方】
商品名・価格・カテゴリ・レビュー評価・レビューコメントを参考にして
「実際に使った人が語っているような自然な文体」で書く。
商品説明文の言葉をそのままコピーしてはいけない。
```

### ✅ Pythonコード（Gemini API呼び出し例）

```python
import google.generativeai as genai
import json

genai.configure(api_key="YOUR_GEMINI_API_KEY")

SYSTEM_PROMPT = """
あなたはThreads（スレッズ）と楽天アフィリエイトのプロフェッショナルです。
[上記のシステムプロンプト全文をここに記述]
"""

def generate_threads_post(item_data: dict, post_type: str = "通常型") -> str:
    """
    楽天APIから取得した商品情報をもとにThreads投稿文を生成する
    
    Args:
        item_data: 楽天APIのレスポンスから抽出した商品情報dict
        post_type: 投稿タイプ（共感型/保存型/セール型/ランキング型/通常型）
    
    Returns:
        生成された投稿文（str）
    """
    
    user_prompt = f"""
【商品情報】
商品名: {item_data.get('itemName', '')}
価格: {item_data.get('itemPrice', '')}円
カテゴリ: {item_data.get('genreId', '')}
評価: {item_data.get('reviewAverage', '')}（{item_data.get('reviewCount', '')}件）
レビュー抜粋: {item_data.get('reviewSummary', '情報なし')}
セール価格: {item_data.get('salePrice', 'なし')}

【投稿タイプ】
{post_type}

【楽天アフィリエイトリンク】
{item_data.get('affiliateUrl', '[リンクをここに挿入]')}

上記の情報をもとに、Threadsの投稿文を生成してください。
"""

    model = genai.GenerativeModel(
        model_name="gemini-1.5-pro",
        system_instruction=SYSTEM_PROMPT
    )
    
    response = model.generate_content(user_prompt)
    return response.text


# 使用例
if __name__ == "__main__":
    # Rakuten APIから取得したサンプルデータ
    sample_item = {
        "itemName": "〇〇シャンプー 500ml",
        "itemPrice": 1980,
        "genreId": "美容・コスメ",
        "reviewAverage": 4.5,
        "reviewCount": 892,
        "reviewSummary": "使った後しっとりする。香りが良い。翌朝の髪がまとまる。",
        "salePrice": None,
        "affiliateUrl": "https://r10.to/xxxxxx"
    }
    
    post = generate_threads_post(sample_item, post_type="共感型")
    print(post)
```

### ✅ 投稿タイプ自動判定ロジック（Pythonコード補足）

```python
def determine_post_type(item_data: dict, is_sale_period: bool = False) -> str:
    """
    商品情報やセール期間に基づいて最適な投稿タイプを自動判定する
    """
    # セール期間中は強制的にセール型
    if is_sale_period and item_data.get('salePrice'):
        return "セール型"
    
    # レビュー件数が多い人気商品はランキング型
    if int(item_data.get('reviewCount', 0)) > 500:
        return "ランキング型"
    
    # 日用品・生活雑貨は保存型
    category = str(item_data.get('genreId', ''))
    daily_categories = ['日用品', '生活雑貨', 'キッチン', '掃除']
    if any(cat in category for cat in daily_categories):
        return "保存型"
    
    # デフォルトは共感型
    return "共感型"
```
